# Integration Guides

Framework and tool guides for using `mermaid-lint` in real authoring
environments.

These docs are meant to answer two questions:

- What is the best setup with the tools that already exist today?
- If that setup is not ideal, what would better first-class support look like?

## Guides

| Guide | What it covers |
|---|---|
| [Astro](astro.md) | Using `mermaid-lint` in Astro content repos, especially through `markdownlint-cli2` and a single `verify` gate |
| [Slidev](slidev.md) | Using `mermaid-lint` in Slidev decks, with current recommendations for editor, CI, and lint workflow |

## Adding a new guide

If you're adding another integration guide, read
[AGENTS.md](AGENTS.md) in this directory first. It defines the structure and
guardrails for future entries like Astro or other Markdown-driven tools.
